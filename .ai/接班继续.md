# 接班继续.md

> 换机器/次日复工时使用此模板恢复工作上下文。

---

## 使用说明

接班时对 AI 说：「接班继续」，AI 将自动读取最近的交接文档并恢复工作状态。

---

## AI 指令

当用户说「接班继续」时，按以下步骤操作：

### 步骤 1: 同步代码
```bash
git pull origin main
```

### 步骤 2: 安装依赖（如有 package.json 变更）
```bash
npm install
```

### 步骤 3: 读取最近的交接文档
查找 `.ai/` 目录下最近的 `交接-*.md` 文件。

### 步骤 4: 恢复工作上下文
向用户汇报：
- 昨日完成的工作
- 进行中的任务
- 需要继续处理的事项
- 注意事项/已知问题

### 步骤 5: 确认环境
```bash
# 检查 Node 版本
node --version

# 检查 PM2 状态（如在服务器上）
pm2 list

# 检查 codegraph 状态
npx codegraph sync
```

### 步骤 6: 开始工作
提示用户可以开始今天的工作，确认是否从接续昨天未完成的任务开始。

---

## 注意事项

- 如果是首次接班（没有历史交接文档），则跳过步骤 3-4，从 git pull 开始
- 如果 git pull 有冲突，优先告知用户解决
- 如果数据库文件（data/mysudoku.db）被 .gitignore 排除，需确认服务器上数据库存在
- 每天第一个活之前必须先接班（如果没有换开发电脑，则继续昨天工作）
